Definitions

Locked vocabulary for the alignment meeting

Task

A discrete actionable item that needs to happen for a trip to be delivered correctly. Has an owner (human or AI), a definition of done, and a closeable lifecycle.

Task-shaped intent

The signal in a client message, ops conversation, or external event that something needs to be done. Examples: client requests a connecting room; vendor announces a price change; flight delayed by 4 hrs.

Origin

The why a task exists — the underlying reason it needs to happen. See §3 for the taxonomy.

Trigger

The originating event that might spawn a task. Triggers feed into AI capabilities or are surfaced directly to a human.

AI task

A task whose execution is performed by AI — either fully (auto tier) or with a human approval gate (assist / escalate tiers).

Human task

A task that requires a human (ops associate, TA, supplier-relationship owner) to execute. Typically relationship-critical, judgement-dependent, or phone-based.

Routing tier

How the task is handled. auto = AI handles end-to-end. assist = AI proposes, human approves. escalate = immediate human attention required.

System signal

An external event detected by ambient monitoring — weather alert, flight delay, vendor pricing change, advisory level. Distinct from a client message.

Trip context

The link between a task and a specific trip, so monitoring signals only fire for active or upcoming trips. A weather alert for Seoul is only a task if a client is travelling to Seoul.

Listening cadence

How often (or how reactively) an AI capability checks for triggers. Per-message, ambient, periodic poll, or daily batch — different per stream.

WRITE stream

A continuous feed of events arriving in our system that may spawn a task. The push side. See §4.

READ stream

A query we run repeatedly against our own data to surface tasks from patterns or absences. The pull side. See §5.
